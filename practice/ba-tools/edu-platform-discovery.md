# 教育培训平台 · Product Discovery

> 日期：2026-07-06 | Phase 2 实战项目
> 状态：Discovery 完成 → 下一步 Plan（轻量架构）

---

## 一、核心问题 & 解决方案

| # | 痛点 | 解决方案 |
|---|------|---------|
| 1 | 课程创建 & 配置流程繁琐 | 整合为统一工作流，Editor 一站式配置 |
| 2 | 课程 ↔ Funding 数据冲突 | **解耦**：课程独立发布，Funding 只标记哪些课可获得资助 |
| 3 | 线下数据无法同步线上 | 提供线上录入通道（出勤率、考试成绩） |
| 4 | 角色权限混乱 | 三层角色隔离：Editor / Trainer / Learner |

## 二、用户角色

| 角色 | 权限范围 | 可见内容 |
|------|---------|---------|
| **Editor** | 全部后台 | 课程配置、Funding、订单管理、消息、财务 |
| **Trainer** | 上课系统 | 被分配的课程、备课、考试配置、批卷 |
| **Learner** | 官网 + 上课系统 | 选课、支付、订单、上课、考试、成绩 |

## 三、关键设计决策

### 1. 课程与 Funding 解耦（核心架构决策）

```
课程发布 ≠ 可获得 Funding

课程管理模块：
  ├── 创建课程 → 配置 → 发布（独立完成，不依赖 Funding）
  └── 状态：draft / published / in_progress / completed

Funding 模块：
  ├── 配置 Funding Agreement（什么条件、哪些课程、什么时段）
  └── 关联课程：标记 course_id → 该课程可获得 Funding
      未关联的课程正常卖，原价出售
```

**解决了之前的状态冲突问题**：课程发布和 Funding 审批是两个独立事件，不需要互相等待。

### 2. Learner 动态定价显示

```
Learner 浏览课程时看到：

┌─────────────────────────────┐
│ 课程：Python 数据分析          │
│ 原价：¥3,000                 │
│ Funding：✅ 可申请             │
│ 你的价格：¥500（Funding 后）    │  ← 系统根据 Learner 信息自动计算
│ [立即购买]                    │
└─────────────────────────────┘

无 Funding 的课程：
┌─────────────────────────────┐
│ 课程：Python 数据分析          │
│ 原价：¥3,000                 │
│ [立即购买]                    │
└─────────────────────────────┘
```

### 3. 向前兼容：Course 模块需预埋的字段

考虑后续模块需求，Course 创建时需预留以下信息点：

| 字段 | 用途 | 后续消费模块 |
|------|------|------------|
| `course_id` | 全局唯一标识 | 全部模块 |
| `course_type` | online / offline / hybrid | 上课系统（决定上课方式） |
| `trainer_id` | 关联教师 | 上课系统（教师视图） |
| `schedule` | 上课时间表 | 上课系统 + Learner 视图 |
| `price_original` | 原价 | 订单系统 + 官网展示 |
| `funding_eligible` | 是否可申请 Funding | Funding 模块 + 官网展示 |
| `tags[]` | 课程标签（方向/级别/技能） | 官网筛选 + 搜索 |
| `exam_config` | 考试方式（线上/线下） | 上课系统 + 考试录入 |
| `status` | draft/published/in_progress/completed | 全部模块 |

## 四、业务主链路（线上）

```
① Editor 配置课程
   → 课程基本信息 + 关联 Trainer + 配置考试方式
   → 发布（独立完成）

② Funding 配置（可并行，不影响课程发布）
   → 创建 Funding Agreement → 关联课程 → 生效

③ Trainer 备课
   → 看到被分配的课程 → 上传课件 → 配置考试内容

④ Learner 选课 & 支付
   → 浏览官网 → 看到价格（原价/Funding后）→ 下单 → 支付

⑤ Learner 上课 & 考试
   → 上课系统 → 按时上课 → 参加考试 → 查看结果

⑥ 线下数据回传
   → Editor/Trainer 录入线下出勤率、考试成绩
```

## 六、模块优先级 & 依赖关系

| 批次 | 模块 | 依赖 | 说明 |
|------|------|------|------|
| **P0** | 课程管理 | 用户中心 | 基石，所有模块都需要 course_id |
| **P1** | 订单 & 支付 | 课程管理 | 有课才能卖 |
| **P1** | Funding | 课程管理 | 关联课程，控制价格 |
| **P2** | 上课系统 | 课程 + 订单 | 有订单才能上课 |
| **P2** | 官网（Learner端） | 课程 + Funding + 订单 | 对外展示 |
| **P3** | 消息系统 | 全部 | 全局通知 |
| **P3** | 线下数据录入 | 上课系统 | 补充通道 |
