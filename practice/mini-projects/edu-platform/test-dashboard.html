🟠 E2E 层 — 模拟真实用户从头到尾完成一个业务目标
测试目标：验证"用户能用这个系统完成一件事"
覆盖模块：官网浏览 · 注册登录 · 课程管理 · Funding · 订单支付 · 上课 · 考试 · 用户管理
运行方式：Playwright 启动真实浏览器 → 自动点击/填表/验证
🟣 集成层 — 验证两个或多个模块之间的数据传递是否正确
测试目标：验证"模块 A 给模块 B 的数据，B 能正确消费"
核心场景：
· 课程管理 → Funding：课程发布后 Funding 能否正确标记 funding_eligible
· Funding → 官网：不同用户看到不同价格（陈同学 ¥500 vs 访客 ¥3,000）
· 订单 → 支付：支付成功后 order.status 是否回写为 paid
· 支付 → 上课：已支付用户才能进入上课系统
· 考试 → 成绩：考试提交后成绩是否正确写入 score 记录
· Auth → 页面：角色切换后页面权限是否正确生效
🔷 组件层 — 验证单个功能点独立工作正常
测试目标：验证"这个按钮/输入框/计时器本身功能正确"
核心场景：
· 考试页面 3 种题型（单选/多选/判断）是否正常渲染
· 倒计时是否启动且格式正确
· 搜索框输入"Python"后课程列表是否过滤
· 课程卡片数量是否正确（≥5 门）
· Funding 表单创建后是否保存成功
· 登录表单输入 → 登录按钮 → 跳转是否正确
· 价格计算：3000 - 2500 = 500（边界值）
· 未登录访客看到的是"登录后购买"而非"立即购买"
