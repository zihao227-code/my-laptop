# 01 - 什么是 LLM（大语言模型）

> 学习日期：2026-07-06

## 一句话理解

**LLM = 一个超级强大的"下一词预测器"**

就像你手机输入法的联想功能，只不过 LLM 读了几乎整个互联网的内容，所以它的"联想"能力异常强大，能达到理解和推理的效果。

---

## 核心概念

### 1. LLM 是什么？

- **L**arge **L**anguage **M**odel（大语言模型）
- 一个在海量文本上训练出来的神经网络
- 能理解、生成、翻译、总结、推理文本

**类比你熟悉的 BA 场景**：
> 把 LLM 想象成一个读过公司所有需求文档、设计稿、会议记录的超级新人——你问它任何问题，它都能基于"读过"的内容给出回答。它不一定会写代码，但它知道代码"长什么样"。

### 2. Token（令牌）— AI 的最小文字单位

- LLM 不直接处理"字"，而是处理 **Token**
- 1 个 Token ≈ 0.75 个英文单词 ≈ 0.5 个中文字
- **为什么重要**：模型的收费、上下文长度限制都以 Token 计算

```
"我是BA" → 大约 4-6 个 Token
"I am a BA" → 大约 4 个 Token
```

### 3. Context Window（上下文窗口）— AI 的"短期记忆"

- 模型一次能"看到"的最大 Token 数量
- Claude 的上下文窗口：200K tokens（约 15 万英文单词 / 一本小说）
- **超出窗口的内容会被"遗忘"**

**BA 类比**：
> 就像你在评审会上，一次只能聚焦讨论有限的需求条目。窗口越大，一次能讨论的内容越多。

### 4. Temperature（温度）— 控制 AI 的"创造力"

- 范围：0 ~ 1（有的模型更高）
- Temperature = 0：严谨、可预测、适合事实性任务
- Temperature = 0.7~1：有创意、多变、适合头脑风暴

| Temperature | 适合的场景 |
|-------------|-----------|
| 0 ~ 0.3     | 写验收标准、数据提取、代码生成 |
| 0.5 ~ 0.7   | 写用户故事、文档起草 |
| 0.8 ~ 1.0   | 头脑风暴、创意方案 |

## 主流 LLM 一览

| 模型 | 开发公司 | 特点 |
|------|---------|------|
| **Claude** (Opus/Sonnet/Haiku) | Anthropic | 安全性、长上下文、推理能力强 |
| GPT-4o / GPT-5 | OpenAI | 多模态（文本+图片+语音）、生态丰富 |
| Gemini | Google | 与 Google 生态深度整合 |
| DeepSeek | DeepSeek | 开源、性价比高 |

当前你在使用的就是 **Claude** 系列模型。

---

## 本节课关键收获

1. LLM 本质上是"预测下一个 token"的机器，但因为训练数据足够大，涌现出了理解和推理能力
2. **Token** 是 AI 的计价单位和记忆单位——理解 Token 才能用好 AI
3. **Context Window** 决定了 AI 一次能处理多少信息
4. **Temperature** 是你控制 AI 输出风格的重要参数
