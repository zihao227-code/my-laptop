# 📦 归档库 — Archive

> 已完成/不再活跃的内容，冷藏而非删除

---

## 什么是"归档"？

- **已完成的项目** — 交付物达成了，资料有价值
- **暂停/放弃的项目** — 暂时不做了但可能有参考意义
- **过时的资源** — 信息已陈旧但历史参考价值
- **已转化的灵感** — 已经变成项目或资产的灵感

---

## 📂 归档分类

```
4-归档库/
├── 已完成项目/   # 按年份/项目名组织
├── 暂停项目/     # 可能重启的项目
├── 过期资源/     # 旧版本文档、废弃技术笔记
└── 历史灵感/     # 已处理过的灵感记录
```

---

## 🔄 归档流程

```
1. 判断：这个内容未来还会用到吗？
   ├─ 会 → 留在当前目录或迁移到资产库
   └─ 不太会，但有历史价值 → 归档

2. 整理：归档前花 5 分钟整理
   ├─ 补充总结（项目复盘/关键收获）
   ├─ 删除明显无用的临时文件
   └─ 保留核心产出和关键决策记录

3. 迁移：移动到对应归档子目录
   └─ 更新相关 [[双向链接]]
```

---

## 💡 归档原则

- **不删除，只冷藏** — 硬盘空间很便宜，灵感可能死灰复燃
- **归档前做复盘** — 留下"为什么开始/为什么结束/学到了什么"
- **保持可检索** — 即使归档了，标签和链接也让它能被搜索到

---

> 🧊 **归档不是终点，是整理后的休眠**
