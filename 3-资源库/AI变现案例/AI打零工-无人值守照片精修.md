# AI 打零工：无人值守照片精修

## 📋 背景

用 AI 在后台无人值守跑副业任务（照片精修等），实现"睡后收入"。

---

## 🎯 变现模式

**类型**：打零工 / 接单（AI 批量处理标准化任务）

核心场景：影楼照片精修、批量图片处理等。

---

## 🔑 核心前提

> 如果 AI 需要你在中间参与、遇到问题会卡住，就不能称之为税后收入。

四个使用场景：

| 场景 | 说明 |
|------|------|
| 白天上班 | AI 后台跑，你做主业 |
| 外出 | AI 留守办公室干活 |
| 睡觉 | 电脑自己干通宵 |
| 多台电脑 | 24 小时轮转 |

---

## 🏗️ 三个工程要点

### 1. 处理五小时重置额度

API 通常 5 小时重置。提前建立"跑满暂停 → 定时重开"的自动机制。

### 2. 总控 + 子任务分派

```
2000 张照片 / 3 个客户 / 9 种风格
    ↓
总控 Agent：拆任务 → 分批次 → 追踪进度
    ↓
子工作流：每批独立跑，互不阻塞
    ↓
本地进度文件：记录完成/失败状态
```

### 3. 跨工作流监督（最关键）

```
每 5 分钟巡检所有在跑工作流
    ↓
检测异常 → 提醒 → 无响应 → 直接干预：
    挂起当前文件 → 强制下一张
    ↓
早上看手机：挂了 5 张，其余处理完毕
    ↓
手机上远程处理挂起的 5 张
```

---

## 🎨 照片精修方法

```
原片 + 客户满意的精修成品 → AI 对比分析差异
    → AI 照着修到和成品一模一样
    → 固化为工作流 → 批量跑
```

> 能不能修得比影楼更好？可以，但没必要。做副业而已，别卷了。

---

## 💡 工程原则

> 绝大多数软件只是从代码层到可视化的冗余。用 AI 执行时，从可视化回归到代码层，它反而更顺手。绕路可视化 = 大量浪费 Token。
