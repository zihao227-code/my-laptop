用户希望将 `practice/mini-projects/edu-platform/` 打造为**完善的起始项目（Starter Project）**。核心思路：与其做很多零散的小练习，不如在一个"足够真实"的项目里，把能遇到的大部分问题都过一遍——遇到问题→解决问题→学到东西。

## 项目定位

- 不追求商业级完善，但追求**场景覆盖面广**
- 覆盖从需求分析、UI 设计、前端开发、测试、Bug 修复、迭代优化的全链路
- 可以作为面试作品/Portfolio

## 希望覆盖的问题类型

| 类别 | 示例 |
|------|------|
| 需求理解 | 用户故事、验收标准、PRD 撰写 |
| UI/交互 | 表单校验、状态流转、错误提示、空状态 |
| 数据处理 | CRUD、搜索过滤、分页、数据一致性 |
| 业务逻辑 | 价格计算、权限控制、状态机、规则引擎 |
| 测试 | 单元测试、集成测试、E2E 测试、回归测试 |
| Bug 修复 | 根因分析、影响范围评估、验证闭环 |
| 工程化 | Git 协作、代码规范、项目结构、模块化 |

**Why:** 用户是零基础学习者，需要在一个具象的项目里反复练习，而不是抽象地学概念。

**How to apply:** 教育平台相关的所有学习和实践都围绕这个项目展开。新增功能时优先选择能带来新学习点的方向。每次遇到新概念（鉴权、路由、状态管理、数据持久化等），都在这个项目里动手实现一遍。
