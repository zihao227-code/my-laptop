用户想系统性地学习 VIBE CODING 相关的**基础概念**，不仅理解"是什么"，还要知道"怎么用"。

## 学习方式

- 每个概念用 BA 能理解的语言解释（[[ba-mentor-guide]]）
- 每个概念在教育平台项目中找到或创造一个实际应用场景
- 先理解概念 → 再动手实现 → 最后总结沉淀为笔记

## 待学概念清单

| 概念 | 一句话解释 | 在教育平台的应用场景 |
|------|----------|-------------------|
| 鉴权 (Authentication) | "你是谁？"——如何验证用户身份 | 登录系统、角色权限 |
| 授权 (Authorization) | "你能干什么？"——不同角色能做什么 | Admin/Editor/Trainer/Learner 权限 |
| 知识库 (Knowledge Base) | AI 的外部"参考书"——让 AI 能查阅特定资料 | 课程内容管理、智能问答 |
| API | 系统之间的"对话方式"——前端怎么和后端/第三方交流 | 支付接口、数据存取 |
| 数据库 | 数据的"仓库"——信息存在哪里、怎么组织 | localStorage、将来可能接真实 DB |
| 状态管理 | 应用"记性"——用户操作后数据怎么变化 | 购物车、登录状态、表单数据 |
| 路由 (Routing) | 页面之间的"导航地图"——URL 和页面的对应关系 | Hash 路由、页面跳转 |
| 部署 (Deploy) | 把代码"发布上线"——让别人也能用 | GitHub Pages 或其他 |
| RAG | "查资料再回答"——AI 先检索相关文档再生成答案 | 智能客服、课程问答 |
| Agent | 能自主使用工具的 AI——不只是聊天，还能做事 | Claude Code 的 Agent 体系 |
| MCP | AI 的"万能插头"——统一的工具连接标准 | 飞书 MCP、数据库 MCP |

## 优先级

1. **先学直接能在项目中用的**：鉴权、路由、状态管理、API
2. **再学理解 AI 产品边界需要的**：RAG、Agent、知识库
3. **最后学工程化相关的**：部署、数据库

**Why:** 用户是 BA 转型 AI PM，不仅需要"用过"这些概念，还需要理解到能跟开发者对话、能判断技术方案的程度。

**How to apply:** 每个概念的学习路径：BA导师用业务类比解释 → 在教育平台中动手实现 → 写学习笔记沉淀。不在抽象概念上花太多时间，优先在实际项目中学习。
