## 一、BA 导师后续需教的设计点

以下是在 Discovery → Plan 过程中发现的、需要深化但暂时搁置的 BA 设计问题：

### 业务流程需要细化的

1. **退款流程**：Learner 支付后取消订单，退款如何处理？Funding 部分是否要退回 Funding 池？
2. **课程变更流程**：已发布的课程改了时间/Trainer，已报名的 Learner 怎么通知？订单是否自动取消？
3. **Funding 审批流程**：Learner 申请 Funding 是自动匹配条件还是人工审核？拒绝后 Learner 还能原价购买吗？
4. **线下考试结果录入流程**：谁录入（Trainer？Editor？）、录入后谁来确认？有争议时走什么流程？
5. **消息触达规则**：哪些事件触发通知？通知发给谁（Learner/Trainer/Editor）？频率上限？

### 设计模式 & 方法论

6. **状态机设计**：Course、Order、Funding 都有多个状态，每个状态的合法流转路径是什么？
7. **权限矩阵**：3 个角色 × 7 个模块 = 21 个权限点的精确设计
8. **异常流程**：支付失败、Funding 过期、课程取消的补偿机制
9. **数据一致性**：线下数据录入后如何和线上数据合并展示

> **教的时候**：用这些真实案例来讲，而不是抽象的"你应该考虑异常流程"。

## 二、技术基础学习清单

用户当前技术基础：零。后续 BA导师 需在合适的时机教以下内容：

| 概念 | 什么时候教 | 教学方式 |
|------|-----------|---------|
| HTML 是什么 | P0 课程管理开始前 | 用"需求文档有标题/段落/表格"来类比 |
| CSS 是什么 | 开始写界面时 | 用"公司 PPT 模板统一风格"来类比 |
| JS 事件是什么 | 按钮要可点击时 | 用"审批流中点击通过后发生什么"来类比 |
| JSON 是什么 | 数据存储时 | 用"Excel 表格转成纯文本格式"来类比 |
| localStorage | 需要保存数据时 | 用"浏览器里的小型 Excel"来类比 |
| API 是什么（后续） | 前后端分离时 | 用"财务系统和课程系统之间怎么传数据"来类比 |

## 三、学习优先级备注

当前核心目标是 **学习 AI Coding**，不是交付生产级系统。因此：

| 可以跳过的 | 应该专注的 |
|-----------|-----------|
| 完整的异常处理 | 核心业务流程跑通 |
| 权限系统的精细实现 | 角色隔离的基本概念 |
| 真实支付对接 | 模拟支付流程 |
| 数据安全 & 加密 | 数据模型和关联关系 |
| 多语言、无障碍 | 中文单语言 |

**原则**：先做出"能演示的"，再优化"能上线的"。

后续做每个模块时，如果某个细节卡住了——先简化，记录下来，回头一起补。
