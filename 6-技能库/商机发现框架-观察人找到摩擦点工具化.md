# 商机发现框架：观察人 → 找摩擦点 → 工具化

## 📌 一句话总结

不是问"什么能赚钱"，而是问"什么人正在做什么事，过程中哪里慢/贵/烦/不确定，能不能用工具帮他吃掉"。

---

## 🔄 核心框架

```
观察人本身 → 理解他们正在做的事 → 看清这件事的目的
                    ↓
            找到过程中的摩擦点
            （慢、贵、烦、难、不确定）
                    ↓
            用工具把摩擦点吃掉
                    ↓
            用户花更少的代价，达成同样的目的
```

---

## 📊 框架验证（所有案例都符合）

| 案例 | 人正在做什么 | 目的 | 摩擦点 | 工具吃掉 |
|------|------|------|------|------|
| 空调订阅 | 反复跑店找空调 | 买到空调 | 不确定性、浪费时间 | 到货通知 = 确定性 |
| 二手车图片 | 挂网上卖车 | 卖掉车 | 照片难看 | AI修图 = 车更值钱 |
| BA写文档 | 开会→写开发能看的文档 | 交付需求 | 写文档耗时 | Agent自动生成 |
| 龙虾APP | 想用AI但装不上 | 用上AI | 安装配对复杂 | 手机一键控制 |
| 影楼精修 | 手动修图 | 交付精修片 | 人工耗时 | AI对标成品自动修 |
| Subscribr | YouTube创作者写脚本 | 出视频 | 写脚本慢 | AI生成脚本 |
| 龙虾开会 | 团队开会复盘 | 对齐认知 | 会议被权力结构扭曲 | AI中立收集+搅拌 |

---

## 🎯 三个验证问题

不是问"这个能赚钱吗"，而是问：

```
① 这些人本来就要做什么事？（行为）
② 做这件事的过程中，哪里慢/贵/烦/不确定？（摩擦点）
③ 能不能做一个工具，帮他们把摩擦点吃掉？（工具化）
```

**关键验证标准**：不是"有没有人需要"，是"是不是已经有人在为这件事花时间/花钱了"。

---

## 💰 工具 vs 消费品 — 两种完全不同的价值逻辑

| | 消费品（洗发露、面包） | 工具 |
|------|------|------|
| **价值在哪** | 产品本身 | 用户通过产品达成的结果 |
| **用完即** | 消耗、满足 | 产出、放大 |
| **好不好怎么判断** | 干净吗？好吃吗？ | 省了多少？多赚了多少？ |
| **付钱买的是** | 产品 | **自己的效率提升** |

> 工具的价值不发生在购买那一刻，发生在用户使用之后产生的结果里。

---

## 📐 工具的定价逻辑

```
用户不用工具时做这件事的成本
  - 用户用工具后做这件事的成本
  = 工具的价值上限

不是：成本 + 利润 = 价格
而是：帮用户省了多少/赚了多少
```

| 案例 | 定价逻辑 |
|------|------|
| 空调订阅 | 用户跑店3天 vs 订阅后直接提货 → 值几块钱 |
| 二手车图片 | 车挂30天卖掉 vs 处理后15天卖掉 → 值几十块 |
| BA工具 | BA写3小时 vs 审核20分钟 → 值多少？ |
