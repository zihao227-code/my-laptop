# RAG 向量数据库设计：PM 视角五层能力模型

## 📌 一句话总结

面试官问"向量数据库怎么设计"，不是考技术，是考你能不能从产品视角设计一整套信息处理系统。技术回答 = 会 AI 的人，PM 回答 = 能做 AI 产品的人。

---

## 🎯 五层能力模型

### 第一层：产品输入能力

| 数据源 | 适配场景 |
|------|------|
| PDF | 企业文档、合同、报告 |
| 网页 | 知识库、帮助中心 |
| 数据库 | 结构化业务数据 |
| 用户行为 | 搜索日志、点击反馈 |

> 这一步决定了产品能不能成立。

### 第二层：数据治理能力

| 治理环节 | 关键问题 |
|------|------|
| OCR | 扫描件做了文字识别吗？ |
| 噪声处理 | 页眉广告、空行、乱码怎么过滤？ |
| 质量过滤 | 低质量文档有没有拦截机制？ |

> 考的是你能不能控制输入质量。

### 第三层：信息结构设计能力（最体现 PM 功底）

> 切片不是技术问题，是产品设计。你在设计**知识如何被理解**。

| 切片策略 | 适用场景 |
|------|------|
| 按段落 | 常规文档 |
| 按语义 | 对话、问答 |
| 按结构 | 合同条款、财报表格 |

关键思维：不是切得够不够细，是切完之后用户能不能搜到答案。

### 第四层：检索系统设计能力

| 维度 | 关键问题 |
|------|------|
| 准确率 | 搜得准不准？ |
| 稳定性 | 每次都一样吗？ |
| 混合检索 | 关键词 + 向量 = 基本能力 |
| 扩展 | 同义词？多语言？实时更新？ |

> 不是能不能搜，而是搜的准不准、稳定不稳定。

### 第五层：闭环优化能力

> 系统上线不是结束，而是开始。

| 指标 | 追踪方式 |
|------|------|
| 召回率 | 该搜到的搜到了吗？ |
| 命中率 | 搜到的用户点了吗？ |
| 用户反馈 | 点完之后满意吗？ |

---

## 🆚 PM 视角 vs 技术视角

| 技术回答 | PM 回答 |
|------|------|
| "用了 Milvus + 混合检索" | 数据从哪来 → 怎么治理 → 怎么切 → 怎么搜得准 → 怎么持续优化 |
| = 会 AI 的人 | = 能做 AI 产品的人 |
