# 主流 AI 模型对比（2026 年 7 月）

## 一句话

选模型就像选外包团队——不是越贵的越好，是**适合你的场景的**才好。

## 四大主流模型

| | Claude (Anthropic) | GPT (OpenAI) | Gemini (Google) | DeepSeek |
|---|---|---|---|---|
| **最强能力** | 长文理解、安全 | 多模态、生态 | 搜索整合 | 性价比 |
| **最大上下文** | 200K tokens | 128K tokens | 200万 tokens | 128K tokens |
| **中文能力** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **价格** | 中高 | 中高 | 中 | **低（约 1/5）** |
| **适合场景** | 需求分析、长文档 | 创意、多模态 | 需要搜索的场景 | 成本敏感、中文场景 |

## 同一模型的不同版本

每个模型都有从小到大的版本，就像同一款车有基础版和顶配版：

| 层级 | Claude | DeepSeek | 适合做什么 |
|------|--------|----------|-----------|
| **小模型**（快+便宜） | Haiku | Flash | 简单翻译、格式转换、关键词提取 |
| **中模型**（平衡） | Sonnet | V4 | 需求分析、文档起草、用户故事生成 |
| **大模型**（强+贵） | Opus | — | 复杂推理、多步骤分析、方案设计 |

## 怎么选？

```
第一步：先试小模型
  → 输出够用吗？→ 够用就用它（最快最便宜）
  → 不够用？→ 往下

第二步：试中模型
  → 输出够用吗？→ 够用就用它（日常工作主力）
  → 还不够？→ 往下

第三步：上大模型
  → 只有复杂推理、多步骤分析才需要
```

## BA 日常工作推荐

| 任务 | 推荐模型 | 理由 |
|------|---------|------|
| 提取功能点 | 小模型 | 结构化输出，小模型足够 |
| 写用户故事 | 中模型 | 需要一定的创造力和格式控制 |
| 分析需求合理性 | 中模型 | 需要推理，但不复杂 |
| 方案对比/决策分析 | 大模型 | 多维度推理，小模型容易漏 |
| 日常聊天/问答 | 小模型 | 越快越好 |
| 翻译/改写 | 小模型 | 纯语言任务 |

## 一个关键认知

你不需要记住每个模型的参数和价格。只需要记住一条铁律：

> **能用小模型解决的，绝不用大模型。**

不是因为省钱——而是因为小模型更快、更稳定、输出更可控。大模型虽然"聪明"，但延迟高、输出有时候反而更飘。
