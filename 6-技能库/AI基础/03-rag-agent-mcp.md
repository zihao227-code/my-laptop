# 03 - RAG、Agent、MCP 概念速览

> 学习日期：2026-07-06

> 📌 本节课是**概念速览**，你不需要完全理解技术细节。
> 目标是知道这些词在说什么、为什么它们重要，后续实践中会逐步加深理解。

---

## 一、RAG（检索增强生成）

### 一句话理解
> **让 AI 先查资料，再回答问题。** 就像考试时让你先翻书再作答。

### 为什么需要 RAG？

AI 有两个天生的限制：
1. **知识截止日期**：模型训练完后，不知道新发生的事情
2. **幻觉**：会编造不存在的"事实"

RAG 的解决思路：**回答前先检索相关文档，只基于检索到的内容回答。**

```
没有 RAG：用户问 → AI 凭记忆回答（可能过时/编造）
有 RAG：  用户问 → 检索知识库 → AI 基于查到的内容回答
```

### BA 场景举例
> 把公司所有的历史需求文档、API 文档导入知识库 → 当你问"这个接口能支持批量导入吗？"→ AI 先去查文档再回答，而不是瞎编。

### 关键组成
| 组件 | 作用 | BA 类比 |
|------|------|---------|
| 知识库 | 存放文档 | 公司的 Confluence/Wiki |
| Embedding | 把文字转成数字以便检索 | 给文档打标签 |
| 检索器 | 找到最相关的文档 | Ctrl+F 的高配版 |
| LLM | 基于检索结果生成回答 | 读懂文档并回答你 |

## 二、Agent（智能体）

### 一句话理解
> **能自主使用工具的 AI。** 普通 AI 只能"说话"，Agent 能"做事"。

### Agent vs 普通 LLM

```
普通 LLM：
  你："帮我查一下今天的天气"
  AI："抱歉，我无法获取实时信息"

Agent（配备了天气查询工具）：
  你："帮我查一下今天的天气"
  AI：调用天气API → 获取数据 → "今天上海晴，22-28°C"
```

### Agent 的核心循环

```
思考 → 行动 → 观察结果 → 再思考 → ...
```

就像一个新人接到任务后：先想怎么办 → 动手做 → 看结果对不对 → 不对再来。

### BA 场景举例
> Agent = 一个能自动打开 Jira 创建工单、发邮件通知、更新 Confluence 文档的 AI 助手。

## 三、MCP（模型上下文协议）

### 一句话理解
> **AI 连接外部工具的"USB 接口标准"。** 就像 USB 让任何设备能插到电脑上。

### 为什么 MCP 重要？

在 MCP 之前，每个 AI 工具连接外部系统都要单独开发：
```
AI → [自定义代码] → GitHub
AI → [自定义代码] → Jira
AI → [自定义代码] → 数据库
```

MCP 之后，一次开发，到处使用：
```
AI ← MCP 协议 → GitHub Server
AI ← MCP 协议 → Jira Server
AI ← MCP 协议 → 数据库 Server
```

### BA 需要知道的
- MCP 是让 AI "能做事"的基础设施
- 你不需要自己开发 MCP Server，只需要知道有这个东西
- 未来公司内部系统（Jira、Confluence、数据库）可能都会提供 MCP 接口

## 三者的关系

```
┌─────────────────────────────────────┐
│              Agent（智能体）          │
│  "我要完成这个任务，需要什么工具？"    │
│                                      │
│  ┌──────────┐  ┌──────────────────┐  │
│  │   RAG    │  │  MCP 工具         │  │
│  │ 查知识库  │  │ 调Jira/发邮件/... │  │
│  └──────────┘  └──────────────────┘  │
│                                      │
│         LLM（大语言模型）              │
│         "理解和推理的大脑"             │
└─────────────────────────────────────┘
```

**简单记忆**：
- **LLM** = 大脑
- **RAG** = 记忆外挂（查资料）
- **Agent** = 有手有脚（能做事）
- **MCP** = 手和工具之间的接口标准

## 关键收获

1. RAG、Agent、MCP 这三个词你会频繁听到——它们是 AI 应用的核心基建
2. 现阶段不要求深入理解，只需要知道"它们是干什么的"
3. 在 Phase 3 做实战项目时，你会亲身体验 Agent 的能力
