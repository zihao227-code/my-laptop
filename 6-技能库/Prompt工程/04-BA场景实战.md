# 04 · BA 三大场景 Prompt 实战

BA 有三个高频场景，写对 Prompt 能直接省一半时间。每个场景给一个可以直接复制用的模板。

## 场景一：从需求文档中提取功能点

**痛点**：拿到几十页的产品需求文档，需要整理成功能清单给开发。纯手工整理至少要半天。

**Prompt 模板**：
```
你是一个有经验的 BA。分析以下需求文档，输出三张表格：

表1 — 功能点清单
| 功能点 | 优先级(P0/P1/P2) | 所属模块 | 一句话说明 |

表2 — 缺失项
| 缺失/不清楚的地方 | 为什么是问题 | 建议怎么补 |

表3 — 接口依赖
| 功能点 | 需要后端提供什么 | 请求/响应格式建议 |

约束：
- 每个功能点的说明不超过 30 字
- P0 = 阻塞核心流程，P1 = 影响体验，P2 = 锦上添花
- 不评价需求好坏，只做客观提取

[粘贴需求文档]
```

## 场景二：生成用户故事

**痛点**：功能点有了，但要写成完整的用户故事 + 验收标准，每条都要 GIVEN-WHEN-THEN。

**Prompt 模板**：
```
你是一个资深 BA。基于以下功能描述，生成用户故事。

每条用户故事包含：
1. 标准格式：作为 [角色]，我想要 [功能]，以便 [价值]
2. 验收标准：至少覆盖正常流程和两种异常情况
3. 验收标准用 GIVEN-WHEN-THEN 格式

要求：
- 每条用户故事后面留一行空白，方便我补充
- 如果有多个角色用到同一个功能，每个角色写一条

功能描述：[粘贴功能点清单]
```

## 场景三：生成流程图

**痛点**：需要画业务流程图给开发看，但 Visio / Draw.io 太慢。

**Prompt 模板**：
```
用 Mermaid flowchart 画出以下业务流程。

流程：[描述流程步骤，用简单的自然语言，比如：
1. 用户打开选课页面
2. 浏览课程列表，可以按类型和标签筛选
3. 点击课程卡片进入详情
4. 点击"立即购买"
5. 如果已登录，创建订单并跳转支付页
6. 如果未登录，跳转登录页面
7. 支付成功后跳转"我的课程"]

要求：
- 用 flowchart TD（从上到下）
- 判断分支用菱形节点 {}
- 正常流程用绿色，异常用红色
- 直接输出完整的 Mermaid 代码块
```

## 通用技巧

1. **先让 AI 做提取，你做判断**。不要问"这个需求好不好"——AI 没有业务背景，判断不准。让它提取功能点、找出缺失项，你来做决策。
2. **一个 Prompt 只做一件事**。不要"帮我分析需求、画流程图、写用户故事"——拆成三个请求，每个的输出都更可控。
3. **把你的隐性知识写进 Prompt**。你知道公司用 Jira 还是飞书、你知道开发团队的接口规范是什么——这些 AI 不知道。写进 Prompt 里，输出立刻从"通用水准"变成"可以直接用的水准"。
