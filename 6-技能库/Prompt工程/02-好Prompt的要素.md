# 02 · 好 Prompt 的四个要素

## 一句话

好 Prompt = **角色 + 任务 + 格式 + 约束**。

## 跟用户故事模板一模一样

你是 BA，你每天都在写这样的东西：

```
作为 [角色]，
我想要 [功能]，
以便 [价值]。
验收标准：
1. GIVEN... WHEN... THEN...
2. ...
```

Prompt 的四个要素，本质上就是同一套思维：

| 用户故事要素 | Prompt 要素 | 例子 |
|------------|------------|------|
| 作为...（角色） | **角色** | "你是一个资深 BA" / "你是一个前端开发" |
| 我想要...（功能） | **任务** | "分析这份需求文档，提取所有功能点" |
| 以便...（价值） | **格式** | "用 Markdown 表格输出，三列：功能名、优先级、说明" |
| 验收标准 | **约束** | "每个功能的说明不超过 20 字，按 P0/P1/P2 排序" |

## 逐个拆解

### 角色

告诉 AI 它是谁。不同的角色，同一个任务给出的输出完全不同。

```
❌ 帮我分析这份需求文档
✅ 你是一个有 10 年经验的资深 BA。帮我分析这份需求文档...
```

### 任务

具体到"做什么"。不是"分析需求"，而是"从需求中提取功能点、标注缺失项、输出清单"。

```
❌ 分析需求文档
✅ 从以下文档中提取所有功能点，标注每个功能点属于哪个模块，
   并指出文档里缺失或没说清楚的地方
```

### 格式

你希望输出长什么样。JSON？Markdown 表格？纯文本？不说的话，AI 随便给你一种。

```
❌ （没指定格式，AI 可能给你一段散文）
✅ 用 Markdown 表格输出，四列：功能点 | 优先级 | 所属模块 | 备注
```

### 约束

字数、风格、禁止什么。这就是验收标准——"什么叫做好了"。

```
❌ （没给约束，AI 可能给你写 2000 字）
✅ 每个功能点的说明不超过 20 字。优先级的判断标准：
   P0 = 阻塞核心流程，P1 = 影响体验，P2 = 锦上添花
```

## 四要素齐全的完整示例

```
你是一个资深 BA。分析以下需求文档，完成三件事：

1. 提取所有功能点，按 P0/P1/P2 分级
2. 标注文档中缺失或没说清楚的地方
3. 指出需要与后端联调的接口

输出格式：Markdown，分三个表格。
约束：每个功能点的描述不超过 30 字。不评价需求好坏，只做提取和标注。
```

## 检查方法

写完 Prompt 之后，问自己：**如果我是收到这份"需求"的人，我能不反问任何问题就直接开始干活吗？**

答案是"能"→ Prompt 合格。答案是"不能"→ 继续补充。
