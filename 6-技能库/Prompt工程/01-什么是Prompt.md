# 01 · 什么是 Prompt

## 一句话

Prompt 就是**你给 AI 写的需求文档**。

## 把它想成你每天都在做的事

你做 BA，每天都在给开发写需求——告诉开发"要做什么、什么格式、有什么约束、怎么验收"。Prompt 完全一样，只不过对象从开发换成了 AI。

| 你写需求 | 你写 Prompt |
|---------|------------|
| 给开发看 | 给 AI 看 |
| 描述功能 | 描述任务 |
| 画原型图 | 给格式要求 |
| 写验收标准 | 写约束条件 |

## 一个对比，你马上就懂了

**❌ 烂 Prompt（像烂需求）**：
```
帮我做个登录页面。
```
问题：没说要什么登录方式、没给异常场景、没指定输出格式。开发拿到这种东西会来反问你，AI 也一样。

**✅ 好 Prompt（像好需求）**：
```
做一个登录页面，HTML+CSS 实现，风格简洁。
要求：
- 功能：邮箱+密码登录，一个登录按钮
- 异常1：邮箱格式不对时，输入框变红，下方显示"邮箱格式不正确"
- 异常2：密码少于 6 位时，同样报错
- 不需要后端，前端模拟即可
```

## 为什么结构化格式对 AI 特别有效

因为 LLM 本质是"模式匹配器"——它读了你前面的话，预测下一个词该是什么。你的 Prompt 越结构化、越少歧义，AI 的输出就越可控。

把这个逻辑反过来想：你写的需求文档，开发最喜欢什么样的？**边界清楚、异常覆盖、格式明确**的。AI 也是。

## 学完这节的行动

打开 Claude Code，把上面的"烂 Prompt"和"好 Prompt"分别试一遍，看输出差多少。
