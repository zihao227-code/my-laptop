# 03 · 三种最常用的 Prompt 技巧

## Zero-shot、Few-shot、Chain-of-Thought

这三个词听起来很技术，其实概念超简单。不需要背，用两次就熟了。

## Zero-shot：不给范例，直接问

就是最普通的 Prompt——你问，AI 答。

```
帮我写一个用户故事的验收标准。
```

AI 凭自己的理解来写。适合**简单、通用性强的任务**。比如"翻译这段话""把这段文字总结成三句话"。

## Few-shot：先给范例，再提问

先给 AI 看 2-3 个你满意的例子，再让它按同样的格式来。

```
以下是两条验收标准的范例：

范例1：
GIVEN 用户已登录且购物车有商品
WHEN 用户点击"提交订单"
THEN 系统创建订单，跳转到支付页，库存扣减

范例2：
GIVEN 用户已登录且购物车为空
WHEN 用户点击"提交订单"
THEN 系统提示"购物车为空"，不创建订单

请按照同样的格式，为以下功能写验收标准：
"用户修改密码"
```

AI 会模仿你给的格式和语气，输出质量明显提升。适合**需要格式一致性**的任务——比如批量生成用户故事、验收标准、PRD 模板。

## Chain-of-Thought (CoT)：让它一步步思考

在 Prompt 末尾加一句"让我们一步步分析"，或者"先列出关键因素，再给结论"。

```
我们的 Q2 用户增长下降了 20%。分析可能的原因，给出建议。

请一步一步来：先列出所有可能的影响因素，对每个因素评估可能性，
再综合给出前三名最可能的原因，最后给出针对性建议。
```

这会让 AI 在推理类任务上准确率大幅提升——因为它被强制要求"不走捷径"，不能跳步骤。

**什么时候用**：分析类任务（根因分析、方案对比、需求拆解）、复杂决策（要不要上某个功能、应该选哪个技术方案）。

## 三者配合

| 场景 | 用什么 |
|------|--------|
| 简单翻译/总结 | Zero-shot 就够了 |
| 批量生成需要格式一致 | Few-shot（给 2-3 个范例） |
| 复杂分析/决策 | CoT（让它一步步推） |
| 复杂 + 格式要求 | Few-shot + CoT 一起用 |
