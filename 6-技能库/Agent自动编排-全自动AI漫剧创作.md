# Agent 自动编排：全自动 AI 漫剧创作 Agent

## 📌 一句话总结

只能跑一遍出结果的系统是自动化流水线，不是 Agent。Agent 的定义性特征在于：**能评估自己的产出、判断哪里不合格、有目的地回滚修改**，而不是动不动全链路重跑。

---

## ❌ 新手常见错误：把管道当 Agent

```
用户输入 → 大纲 → 剧本 → 分镜 → 图片 → 视频
```

三大致命问题：

| 问题 | 表现 | 根因 |
|------|------|------|
| 信息漂移 | 每个环节各自读原始输入，越往后越偏 | 没有唯一事实来源 |
| 角色一致性崩溃 | 第一帧红衣，第十帧蓝衣 | 生成模型无状态，没有注入状态记忆 |
| 无法局部修改 | 第三段有问题 → 全链路重跑 | 没有模块化回滚点 |

---

## 🏗️ 三层架构设计

### 第一层：唯一事实来源

**需求解析 Agent** — 在任何工作开始前，把用户模糊输入转成结构化需求文档。

```
用户："帮我做都市休闲短剧，男主未婚，觉醒系统，第一集要有爽点"
        ↓
结构化文档：
├── 题材：都市 + 系统流
├── 平台：默认竖屏短剧
├── 目标受众：18-30岁
├── 核心钩子：第一集爽点
├── 视觉风格：写实（默认）
├── 强制要求：男主未婚、第一集爽点
└── 硬性禁忌：无
```

> 后续所有 Agent 只读这份文档，不对原始输入做二次解释。

### 第二层：模块化流水线

```
需求解析 → 故事大纲 → 角色设定 → 场景设定
    → 剧本生成 → 分镜拆解 → 提示词构建
    → 图片生成 → 视频生成 → 配音字幕合成
```

**关键设计 — 角色卡 + 场景卡 = 人为注入的状态记忆**：

> 生成模型本身是彻底无状态的。上一张的结果不会自动变成下一张的先验。

角色卡承载：外貌、着装、身高体型。
场景卡承载：环境描述、光线、色调。

统一提示词模板 + 约束强注入每次生成请求 → 几十个镜头间维持视觉一致性。

**分镜拆解是压力最大、最被低估的节点**：剧本切分成镜头单元，每个单元只承载一个画面意图（情节/机位/动作/情绪/角色位置关系）。

### 第三层：评估闭环 ← Agent 和管道的分水岭

**专用 Evaluator**（输出结构化检查项，不是主观评价）：

| Evaluator | 审查内容 |
|------|------|
| Script Evaluator | 前三秒有没有钩子？故事有有效冲突？对白密度过高？ |
| Storyboard Evaluator | 镜头描述过于抽象？图像模型能理解吗？ |

**回滚逻辑**：

```
评估不通过 → 定位问题节点 → 只触发上游对应环节重新生成

"钩子强度不达标" → 剧本层 → 重跑剧本+下游
"第七镜头角色关系模糊" → 分镜层 → 重跑分镜+下游

不是：动不动全链路重跑（这才是 Demo 和产品的分界线）
```

---

## 🔑 Agent vs 自动化管道的本质区别

| 自动化管道 | Agent |
|------|------|
| 一条无状态管道，跑完即止 | 跑完 → 评估 → 判断 → 回滚修改 |
| 出问题只能全链路重跑 | 定位到具体节点，局部重跑 |
| 没有质量门控 | 每个关键环节有 Evaluator 把关 |
| 把模型串起来 = Agent | 模型之外加上架构层 = Agent |
