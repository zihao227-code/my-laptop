# 01 · 编程基础速览（BA 视角）

## 不用担心

你不是来学写代码的。你是来学到**能和开发对话的程度**。读完这篇，你能理解 HTML/CSS/JS/JSON/API 这些词到底在说什么——不需要写一行代码。

## HTML = 需求文档的结构

HTML 管的是"页面上有什么"——标题、段落、按钮、输入框、表格。就像你写需求文档时定义"这个页面有搜索框、有列表、有提交按钮"一样，HTML 就是把这句话翻译成了浏览器能懂的语言。

| 你写需求 | HTML 对应 |
|---------|----------|
| "页面顶部有一个大标题" | `<h1>` |
| "下面有一段说明文字" | `<p>` |
| "有一个搜索框" | `<input>` |
| "有一个提交按钮" | `<button>` |
| "下面是一个数据表格" | `<table>` |

**一句话**：HTML 是骨架。就像你的需求文档的目录结构。

## CSS = 公司的 PPT 模板

CSS 管的是"页面上东西长什么样"——颜色、大小、间距、字体。就像你们公司统一用的 PPT 模板——同样的内容，换上不同的模板，视觉效果完全不同。

| PPT 模板 | CSS |
|---------|-----|
| 统一标题字体和颜色 | `font-family`, `color` |
| 统一背景色 | `background` |
| 统一行间距 | `padding`, `margin` |
| 统一按钮样式 | `border-radius`, `box-shadow` |

**一句话**：CSS 是皮肤。同一份 HTML，换上不同的 CSS，可以完全变一个风格。

## JavaScript = 审批流自动规则

JS 管的是"页面上东西会怎么动"——点击按钮发生什么、输入框输错了怎么提示、数据怎么保存。就像你在 OA 系统里设的审批规则——"金额大于 5000 必须总监批""通过后自动发通知"。

| 审批规则 | JavaScript |
|---------|-----------|
| "金额>5000 须总监批" | `if (amount > 5000) { ... }` |
| "通过后自动发通知" | `save() + toast('成功')` |
| "驳回须填原因" | `if (!reason) { showError() }` |

**一句话**：JS 是肌肉。它让页面动起来。

## JSON = Excel 转成纯文本

JSON 是一种"用纯文本表示数据的格式"。你 Excel 里的一行一行数据，转成 JSON 长这样：

```json
{
  "课程名称": "Python 数据分析",
  "价格": 3000,
  "讲师": "李老师",
  "标签": ["Python", "数据", "初级"]
}
```

**一句话**：JSON 是系统之间传数据的"普通话"——前端和后端都懂。

## API = 系统之间的"对话接口"

API 是前端和后端之间的约定——"你要什么数据，我返回什么数据"。就像你给开发提需求时说的"前端传一个 user_id，后端返回这个用户的所有订单"。

```
前端：GET /api/orders?user_id=U005
后端：{ "orders": [ {...}, {...} ] }
```

**一句话**：API 就是"我给你什么，你还我什么"的契约。

## 三者怎么配合

```
用户看到 → HTML（骨架）
     ↓
   好看吗 → CSS（皮肤）
     ↓
   点按钮 → JavaScript（肌肉）
     ↓
   要数据 → API（对话）
     ↓
   数据格式 → JSON（普通话）
```
