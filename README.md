## 📐 设计理念

- **PARA 方法**：项目 / 领域 / 资源 / 归档，辅以灵感和技能维度
- **原子化笔记**：一篇笔记只讲一个概念，用 `[[双向链接]]` 织成网络
- **渐进式迭代**：先记录再整理，先数量再质量，持续演化
- **Git 同步**：版本控制 + 多地备份，每一次修改都有迹可循

## 🗂️ 目录结构

| 目录 | 用途 | 节奏 |
|------|------|------|
| [1-项目库](./1-项目库/) | 有明确目标和截止日期的短期任务 | 项目结束即归档 |
| [2-资产库](./2-资产库/) | 长期积累的可复用资产：模板、脚本、配置 | 持续沉淀 |
| [3-资源库](./3-资源库/) | 外部资料整理：文章、教程、工具、书摘 | 随时补充 |
| [4-归档库](./4-归档库/) | 已完成/不再活跃的内容，冷藏而非删除 | 定期迁移 |
| [5-灵感库](./5-灵感库/) | 一闪而过的想法、待验证的假设 | 随时捕获 |
| [6-技能库](./6-技能库/) | 技术栈、方法论、操作手册、踩坑记录 | 持续打磨 |

```bash

# 2. 用 Obsidian 打开
```
